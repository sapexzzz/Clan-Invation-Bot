# Telegram Bot Invite Clan

Telegram-бот на Python для приёма заявок в клан Standoff 2 с модерацией через Telegram-группу.

## Возможности

- Подача заявки через пошаговый FSM-сценарий
- Проверка минимальных требований:
  - KD >= 1.0
  - Часы >= 50
- Автоматическое отклонение неподходящих заявок
- Защита от спама: не чаще 1 заявки в 10 минут
- Блокировка повторной подачи, если уже есть активная заявка (pending)
- Отправка новой заявки в админ-группу с inline-кнопками
- Проверка прав администратора перед каждым действием
- Создание одноразовой invite-ссылки на 24 часа после одобрения
- **Блокировка пользователей**: кнопка `🚫 Заблокировать` в сообщении заявки
- **Middleware**: заблокированный пользователь получает отказ на любое сообщение или нажатие кнопки
- **ClanPanel**: терминальная утилита для управления пользователями и просмотра заявок
- Хранение данных в SQLite через aiosqlite

## Настройка

Создай файл `.env` в корне проекта:

```env
BOT_TOKEN=your_bot_token
GROUP_ID=-1001234567890
TOPIC_ID=968
```

### Переменные окружения

- `BOT_TOKEN` — токен бота от BotFather
- `GROUP_ID` — ID группы или форум-группы, куда приходят заявки
- `TOPIC_ID` — ID темы в форум-группе, опционально

## Права бота

Бот должен быть администратором группы и иметь права:

- отправлять сообщения
- создавать invite link

## Запуск

```bash
.venv/bin/python main.py
```

## Как это работает

1. Пользователь нажимает кнопку подачи заявки.
2. Бот собирает данные по шагам: игровой ID, ник, ранги, часы, KD.
3. Если часы меньше 50 или KD вне допустимого диапазона, заявка автоматически отклоняется.
4. Если данные подходят, бот показывает подтверждение и даёт возможность изменить любое поле.
5. После отправки заявка сохраняется в SQLite и публикуется в админ-группе.
6. Администратор принимает, отклоняет или **блокирует** пользователя через inline-кнопки.
7. При принятии пользователю отправляется персональная invite-ссылка сроком на 24 часа.
8. Заблокированный пользователь получает отказ на любое взаимодействие с ботом.

## ClanPanel

Терминальная утилита для управления пользователями без запуска бота:

```bash
.venv/bin/python clanpanel.py
```

| Пункт | Описание |
|---|---|
| `1` | Список всех пользователей: TG ID, ник, статус блока, счётчики заявок |
| `2` | Заблокировать пользователя по Telegram ID |
| `3` | Разблокировать пользователя по Telegram ID |
| `4` | Список заявок с фильтром по статусу |
| `0` | Выход |

## База данных

Используются таблицы:

- `users` — хранит `telegram_id`, время последней заявки и флаг `is_blocked`
- `applications` — хранит данные анкеты и статус заявки

## Статусы заявок

- `pending`
- `accepted`
- `rejected`
- `rejected_auto`

## Примечания

- Для форум-группы используется `TOPIC_ID` как `message_thread_id`.
- Если `TOPIC_ID` не задан, сообщение отправляется в общий чат.
- Для локального состояния FSM используется `MemoryStorage`.
- Миграция БД (добавление `is_blocked`) применяется автоматически при запуске бота и `clanpanel.py`.
